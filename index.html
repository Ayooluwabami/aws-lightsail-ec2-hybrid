Request lifecycle end-to-end private proxy

  • 1 User hits Lightsail — public IP, HTTPS. Nginx terminates TLS.
  • 2 Static asset? Served directly from Lightsail SSD (images, CSS, JS).
  • 3 Dynamic path → proxy_pass to EC2 private IP 10.0.1.x:80 via VPC peering.
  • 4 EC2 (Apache) processes the request, returns response to Lightsail over the same link.
  • 5 Outbound bandwidth billed at Lightsail bundle rate — EC2 egress cost essentially zero.
